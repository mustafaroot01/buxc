# دليل مزامنة وفلترة الطلاب (Mobile API Guide)

هذا الدليل مخصص لمبرمج تطبيق الموبايل ليشرح له كيفية جلب قائمة الطلاب الكاملة وكيفية التعامل مع البيانات المرجعة للقيام بالفلترة داخل التطبيق.

---

## 1. رابط الـ API (Endpoint)

لجلب قائمة الطلاب، نستخدم طلب من نوع `GET`:

**النطاق الرئيسي:** `https://bucx.diyala.net`
**الرابط:** `/api/v1/teacher/students`

---

## 2. البارامترات المطلوبة للمزامنة الكاملة

لضمان جلب **جميع الطلاب** دون استثناء (أول مرة)، يجب إرسال البارامترات التالية:

| البارامتر | القيمة | الوصف |
| :--- | :--- | :--- |
| `all` | `1` | يجبر السيرفر على إرسال كل الطلاب التابعين للمعلم. |
| `per_page` | `5000` | يحدد عدد السجلات في الطلب الواحد (لضمان جلب الـ 101 طالب معاً). |

**مثال للرابط الكامل:**
`https://bucx.diyala.net/api/v1/teacher/students?all=1&per_page=5000`

## 4. كيفية الفلترة داخل تطبيق الموبايل (Local Filtering)

بدلاً من طلب البيانات من السيرفر في كل مرة يغير فيها المعلم "الكروب"، يفضل أن يقوم التطبيق بجلب القائمة الكاملة مرة واحدة (Sync) وتخزينها محلياً (مثلاً في SQLite أو Room)، ثم الفلترة كالتالي:

### أ. الفلترة حسب المرحلة (Filter by Stage):
عندما يختار المعلم "المرحلة الأولى"، يقوم التطبيق بعرض الطلاب الذين يملكون:
`student.stage_name == "المرحلة الأولى"`

### ب. الفلترة حسب الكروب (Filter by Group):
عند اختيار كروب معين:
`student.group_name == "الكروب A"`

## 5. مثال عملي للمبرمج (Logic Example)

لنفترض أن المعلم اختار الفلاتر التالية في التطبيق:
- **المرحلة:** المرحلة الثالثة
- **الكروب:** كروب B
- **نوع الدراسة:** مسائي

يجب أن يكون منطق الفلترة (Pseudo Code) كالتالي:

```javascript
// مصفوفة الطلاب التي تم جلبها من السيرفر وتخزينها محلياً
const allStudents = data.students; 

// القيم التي اختارها المعلم من واجهة التطبيق (UI)
const selectedStage = "المرحلة الثالثة";
const selectedGroup = "كروب B";
const selectedStudyType = "مسائي";

// عملية الفلترة الذكية
const filteredList = allStudents.filter(student => {
    return student.stage_name === selectedStage &&
           student.group_name === selectedGroup &&
           student.study_type === selectedStudyType;
});

// النتيجة: عرض filteredList فقط في القائمة أمام المعلم
console.log(`تم العثور على ${filteredList.length} طالب مطابق للفلاتر`);
```

### 💡 لماذا هذه الطريقة هي الأفضل؟
1. **السرعة:** الفلترة تتم في أجزاء من الثانية لأن البيانات موجودة مسبقاً في ذاكرة الموبايل.
2. **العمل أوفلاين:** يمكن للمعلم تغيير الفلاتر والبحث عن الطلاب حتى لو انقطع الإنترنت.
3. **توفير البيانات:** لا نحتاج لإرسال طلب جديد للسيرفر في كل مرة يغير فيها المعلم خياراً بسيطاً.

---

## 6. نصيحة للمزامنة الدورية (Incremental Sync)
بعد المزامنة الأولى (التي استخدمنا فيها `all=1`)، يجب على التطبيق حفظ قيمة `sync_version` المستلمة.
في المرة القادمة، يطلب التطبيق التحديثات فقط بإرسال:
`GET /api/v1/teacher/students?since_version=12`

السيرفر سيعيد فقط الطلاب الذين تغيرت بياناتهم أو تمت إضافتهم بعد هذا الإصدار.

---

**ملاحظة:** يجب إرسال الـ `Authorization: Bearer {token}` في الـ Header الخاص بالطلب.
