# مرجع هيكلية قاعدة البيانات (Database Schema) 📊

هذا الملف يحتوي على أسماء الحقول الدقيقة وأنواعها للجداول الأساسية المستخدمة في الـ API.

---

## 1. جدول المراحل الدراسية (`academic_stages`)
| اسم الحقل | النوع | الوصف |
| :--- | :--- | :--- |
| `id` | `varchar (UUID)` | المعرف الفريد للمرحلة |
| `name` | `varchar` | اسم المرحلة (مثلاً: الأولى، الثانية) |
| `status` | `boolean` | حالة المرحلة (1 نشط، 0 غير نشط) |

---

## 2. جدول المجموعات (`academic_groups`)
| اسم الحقل | النوع | الوصف |
| :--- | :--- | :--- |
| `id` | `varchar (UUID)` | المعرف الفريد للمجموعة |
| `name` | `varchar` | اسم المجموعة (مثلاً: A، B، C) |
| `stage_id` | `varchar (UUID)` | يربط المجموعة بالمرحلة |
| `study_type` | `varchar` | نوع الدراسة: **`morning`** أو **`evening`** |

---

## 3. جدول الطلاب (`students`)
| اسم الحقل | النوع | الوصف |
| :--- | :--- | :--- |
| `id` | `varchar (UUID)` | المعرف الفريد للطالب |
| `student_external_id` | `varchar` | رقم الهوية أو الرقم الجامعي للطالب |
| `first_name` | `varchar` | الاسم الأول |
| `second_name` | `varchar` | اسم الأب |
| `last_name` | `varchar` | اللقب / الكنية |
| `group_id` | `varchar (UUID)` | يربط الطالب بمجموعة معينة |
| `qr_payload` | `varchar (24)` | الرمز الفريد للـ QR Code الخاص بالطالب |
| `photo_path` | `varchar` | مسار الصورة الشخصية للطالب |

---

## 4. جدول المحاضرات (`lectures`)
| اسم الحقل | النوع | الوصف |
| :--- | :--- | :--- |
| `id` | `varchar (UUID)` | المعرف الفريد للمحاضرة |
| `title` | `varchar` | عنوان المحاضرة |
| `subject_id` | `varchar (UUID)` | يربط المحاضرة بالمادة الدراسية |
| `teacher_id` | `varchar (UUID)` | يربط المحاضرة بالأستاذ |
| `group_id` | `varchar (UUID)` | يربط المحاضرة بالمجموعة المستهدفة |
| `start_time` | `datetime` | وقت وتاريخ بداية المحاضرة |
| `status` | `varchar` | حالة المحاضرة: **`active`** (مفتوحة) أو **`closed`** (مغلقة) |

---

## 5. ملاحظات للمبرمج:
- جميع المعرفات (`ID`) هي بتنسيق **UUID** (نص طويل فريد).
- حقل `study_type` يقبل قيمتين فقط: `morning` للدراسة الصباحية و `evening` للدراسة المسائية.
- حقل `status` في المحاضرات هو نص (`active` / `closed`).
